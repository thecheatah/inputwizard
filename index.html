Input Wizard Source Code Documentation

About

This document will talk about the how and why this project is architected the way it is. The input wizard framework was architected by Ravneet Singh.

General Concepts

The input wizard framework uses the following key design patterns:
  • Dependency Injection/Inversion of control
    The Form Widget is responsible for implementing the IoC. It looks for widgets which implement specific interfaces and ties them together with the rest of the system use these interfaces or abstract classes.
    We use the following Interfaces or Abstract Classes:
    • DataErrorChangeDelegate
    • DataValueChangeDelegate
    • DisplayAreaVisibilityListener
    • DisplayModeListener
    • Field
  • Delegation Model to decouple components
